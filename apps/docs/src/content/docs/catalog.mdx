The factory's enterprise catalog — **363 agents across five departments** — laid
out as a periodic table. Each tile is one agent, positioned in its **domain
group** and colored by domain. The **layer dot** marks how it is built (OOTB →
Agent Designer → Custom ADK → Data Agent), the **corner icon** marks its trigger
(event · chat · scheduled), and a **shield** marks a human-in-the-loop
checkpoint. Switch departments with the tabs; hover or focus a tile for a quick
read, or click through to that agent's spec. Each agent page renders the
agent's full **Open Knowledge Format (OKF)** bundle — playbook, tools, source
systems, data entities, workflow, and evals — compiled from the same spec the
factory builds the agent from.

<PeriodicTable />

:::tip[Industry verticals]
The grid above is the horizontal half of the fleet. The **industry
verticals** — retail, banking, insurance, telco, and manufacturing agents
organized by value stream and persona — have their own
[periodic table](/ge-agent-factory/catalog-verticals/). To slice both halves
together, use the [catalog explorer](/ge-agent-factory/catalog/explorer/): a
faceted view of the whole fleet filterable by industry, business function,
and value stream, with function-family links (filtering by *procurement*
also surfaces the retail and manufacturing supply-chain agents that
specialize it).
:::

:::note[Where this comes from]
This grid is generated from the same source of truth the factory itself
consumes — the use-case catalog and the drift-gated agent-spec registry
(`apps/factory/src/agent-spec-registry.generated.json`). Nothing here is
hand-maintained: `apps/docs/scripts/gen-catalog.mjs` rebuilds it on every docs
build, so the catalog can never drift from the specs.
:::
