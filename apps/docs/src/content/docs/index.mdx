## Get to your first proof

<Tabs>
  <TabItem label="Ask your coding agent">

Give your coding assistant the factory skills, then ask it to install and verify the repository. This path requires Bun because Bun runs the installer.

```bash title="from the directory that should contain the checkout"
curl -fsSL https://raw.githubusercontent.com/vamsiramakrishnan/ge-agent-factory/main/packages/create-ge-agent-factory/bin/create-ge-agent-factory.mjs \
  | bun - -- --yes --skills agents
```

The [local setup guide](/ge-agent-factory/start/getting-started/) also covers Claude Code, Gemini CLI, Codex, Antigravity, and MCP clients.

  <TabItem label="Drive it yourself">

Everything here is local and credential-free. The result is a validated workspace on disk with its contract, generated code, tools, fixtures, evaluations, and proof artifacts.

<div class="ge-journey">
  <CardGrid>
    <Card title="Set up" icon="setting">
      <span class="ge-label">Step 01</span> <span class="ge-eta">~5–10 min · once</span>

      ```bash
      mise run setup
      ```

      Provisions the pinned toolchain, repository dependencies, skills, and the `ge` CLI.
    </Card>
    <Card title="Initialize" icon="document">
      <span class="ge-label">Step 02</span> <span class="ge-eta">~30 sec</span>

      ```bash
      ge init
      ```

      Discovers local configuration and writes `.ge.json`. Bare `ge` then shows the current state and next action.
    </Card>
    <Card title="Prove" icon="approve-check">
      <span class="ge-label">Step 03</span> <span class="ge-eta">~5 min</span>

      ```bash
      ge prove
      ```

      Compiles and validates one starter contract, then prints the workspace and proof paths.
    </Card>
  </CardGrid>
</div>

  </TabItem>
  <TabItem label="Use the console">

Open the browser control room when you want to capture a contract, follow a run, inspect proof, or repair a blocker visually.

```bash
mise run console
```

The console opens at `http://localhost:18260`.

## What a local proof leaves behind

<CardGrid>
  <Card title="Readable contract" icon="document">
    Role, scope, systems, tools, refusal rules, success criteria, and evidence requirements stay reviewable before and after generation.
  </Card>
  <Card title="Generated workspace" icon="seti:python">
    Google Agent Development Kit code, tool adapters, fixtures, and support code live in a normal project you can inspect.
  </Card>
  <Card title="Source-system twins" icon="seti:database">
    Local simulations model the APIs, data, and failure modes the agent must handle without touching production.
  </Card>
  <Card title="Proof pack" icon="approve-check">
    Evaluations, traces, stage verdicts, and promotion evidence explain what passed and what still needs repair.
  </Card>
</CardGrid>

:::note[One registry, three operator surfaces]
The CLI, console, and MCP tools dispatch through the same capability registry and result contracts. Choose the surface that fits the task; the underlying command remains the same.
:::
