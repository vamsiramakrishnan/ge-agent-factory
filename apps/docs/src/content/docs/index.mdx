## Start with skills — let your coding agent install and run the factory

The fastest path is not a manual checklist. GE Agent Factory ships as agent
skills, including the installer, so Claude Code, Gemini CLI, Codex,
Antigravity, agents-cli-style agent sessions, or any MCP-capable assistant can
bootstrap the repo and then operate the factory with the same playbooks the
harness uses.

```bash
bunx create-ge-agent-factory        # clone + guided, verified install
```

| Surface | Skills-first setup |
| --- | --- |
| Claude Code | `/plugin marketplace add vamsiramakrishnan/ge-agent-factory`, then `/plugin install factory-bootstrap@ge-agent-factory` |
| Gemini CLI | `gemini extensions install https://github.com/vamsiramakrishnan/ge-agent-factory` |
| Codex · Antigravity · agents-cli-style sessions | `bunx create-ge-agent-factory --skills agents` |
| Any MCP client | Run `bun tools/mcp-server.mjs` from a checkout to expose the `factory_*` tools |

The generated workspaces still hand off to
[Google agents-cli](https://google.github.io/agents-cli/) / ADK / Gemini
Enterprise. Skills are the operator layer that teaches your assistant how to
set up and drive this factory.

## The manual fallback in three commands

Everything below runs on your machine — no cloud credentials required. Use it
when you want to drive setup yourself instead of asking an agent to follow the
skills.

<div class="ge-journey">
  <CardGrid>
    <Card title="1 · Set up" icon="setting">
      <span class="ge-eta">~5–10 min · one time</span>

      ```bash
      mise run setup
      ```

      Provisions the toolchain and puts the `ge` CLI on your PATH.
    </Card>
    <Card title="2 · Initialize" icon="document">
      <span class="ge-eta">~30 s</span>

      ```bash
      ge init
      ```

      Discovers configuration, writes `.ge.json`. Bare `ge` now shows a
      status board with the next step.
    </Card>
    <Card title="3 · Compile and prove" icon="approve-check">
      <span class="ge-eta">~5 min</span>

      ```bash
      ge devex smoke
      ```

      One contract compiled into a validated agent workspace: real ADK code,
      tools, simulations, evals, and the proof artifacts — on disk, inspectable.
    </Card>
  </CardGrid>
</div>

From there: `ge agents build --canary` compiles a contract to the build
boundary, the console (`mise run console`) shows every run live, and
`ge agents ship` hands the proven workspace to agents-cli → Agent Engine →
Gemini Enterprise, in your own Google Cloud project.
