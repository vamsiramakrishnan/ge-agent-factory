## Choose your fastest path

<Tabs>
  <TabItem label="Ask your coding agent">

The polished path is skills-first: give your assistant the factory playbooks and let it install, verify, operate, and repair the repo with the same instructions used by the harness.

```bash
bunx create-ge-agent-factory --skills agents
```

| --- | --- |
| Claude Code | `/plugin marketplace add vamsiramakrishnan/ge-agent-factory`, then `/plugin install factory-bootstrap@ge-agent-factory` |
| Gemini CLI | `gemini extensions install https://github.com/vamsiramakrishnan/ge-agent-factory` |
| Codex · Antigravity · agents-cli-style sessions | `bunx create-ge-agent-factory --skills agents` |
| Any MCP client | `bun tools/mcp-server.mjs` exposes the `factory_*` tools from a checkout |

  <TabItem label="Drive it yourself">

Everything below is local and credential-free. It gives you a validated canary workspace on disk: contract, generated ADK code, tools, fixtures, evals, and proof artifacts.

<div class="ge-journey">
  <CardGrid>
    <Card title="Bootstrap" icon="setting">
      <span class="ge-label">Stage 01</span> <span class="ge-eta">~5–10 min · one time</span>

      ```bash
      mise run setup
      ```

      Provisions Bun, Python, uv, Terraform, repo dependencies, and the `ge` CLI.
    </Card>
    <Card title="Initialize" icon="document">
      <span class="ge-label">Stage 02</span> <span class="ge-eta">~30 s</span>

      ```bash
      ge init
      ```

      Discovers local configuration and writes `.ge.json` so bare `ge` can show a next-step status board.
    </Card>
    <Card title="Prove" icon="approve-check">
      <span class="ge-label">Stage 03</span> <span class="ge-eta">~5 min</span>

      ```bash
      ge devex smoke
      ```

      Compiles one contract into a validated workspace and refreshes the proof that gates handoff.
    </Card>
  </CardGrid>
</div>

## One layered system

Three operator surfaces sit on one command registry; the registry drives one
generator engine; the engine leaves a contract, an ADK workspace, and a proof
pack on disk. Only the final handoff crosses into your Google Cloud project.

<p align="center">
  <img src="/ge-agent-factory/assets/diagrams/platform-stack.svg" alt="Five stacked layers: the surfaces layer (ge CLI, console, MCP tools and skills) drives the operator core (one command registry and job runner), which drives the generator engine (contract compiler, workspace generator, twins and synthetic data, eval compiler); the engine's output lands on disk as the contract, the ADK workspace, and the proof pack; a thick handoff edge crosses into your Google Cloud project, where agents-cli and ADK deploy to Agent Engine and publish to Gemini Enterprise" width="640" loading="lazy" />
</p>

:::note[One registry, three surfaces]
Every operator command is a CLI command, an HTTP route in the console, and an MCP tool backed by the same registry and job runner. Start in whichever surface fits your workflow; the behavior is the same.
:::
