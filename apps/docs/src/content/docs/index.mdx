## Your first agent, three commands

Everything below runs on your machine — no cloud credentials required.

<div class="ge-journey">
  <CardGrid>
    <Card title="1 · Set up the toolchain" icon="setting">
      <span class="ge-eta">~5–10 min · one time</span>

      ```bash
      mise run setup
      ```

      Provisions Bun, Python/uv, and Terraform, installs the `ge` CLI onto
      your PATH, and starts the daemon.
    </Card>
    <Card title="2 · Initialize" icon="document">
      <span class="ge-eta">~30 s</span>

      ```bash
      ge init
      ```

      Discovers your configuration and writes `.ge.json`. Bare `ge` now shows
      a status board with the next step.
    </Card>
    <Card title="3 · First canary agent" icon="rocket">
      <span class="ge-eta">~5 min</span>

      ```bash
      ge devex smoke
      ```

      Doctor → one generated, validated canary workspace: local proof that
      the whole pipeline works.
    </Card>
  </CardGrid>
</div>

From there the loop is **build → watch → ship**: build agents, watch every
run live in the console (recorded and replayable, event by event), then ship
to the cloud when you choose. `mise run console` opens the operator UI —
Overview · Pipeline · Interview · Fleet · Repair Queue · Runs · Readiness.

:::note[One registry, three surfaces]
Every `ge` command is a CLI command, an HTTP route (the console), and an MCP
tool — one operator registry, dispatched through one job runner, so the three
surfaces can never disagree. See [Console & APIs](/ge-agent-factory/reference/console-and-apis/).
:::
