## The factory path

Every agent takes the same inspectable line — each stage writes artifacts you
can open, diff, and gate on.

<CardGrid>
  <Card title="1 · Interview or choose a use case" icon="comment">
    Start from business intent, not an implementation guess. The interview
    writes the spec for you. [Author a spec →](/ge-agent-factory/cookbooks/author-a-spec-via-interview/)
  </Card>
  <Card title="2 · Materialize the spec" icon="document">
    The spec is the contract: systems, entities, tools, workflow, tests, and
    runtime expectations. [Specs and OKF →](/ge-agent-factory/concepts/specs-and-okf/)
  </Card>
  <Card title="3 · Generate the workspace" icon="setting">
    Real ADK code, tools, fixtures, simulator bindings, smoke tests, eval
    config, and a grounding bundle. [Generate an agent →](/ge-agent-factory/cookbooks/generate-an-agent/)
  </Card>
  <Card title="4 · Validate locally" icon="approve-check">
    Generation → validation → preview runs entirely on your machine, no cloud
    credentials required. [The factory line →](/ge-agent-factory/concepts/the-factory-line/)
  </Card>
  <Card title="5 · Ship remotely" icon="rocket">
    Deploy Agent Runtime, register MCP tools, publish to Gemini Enterprise,
    and verify live access — in **your** GCP project. [Operations →](/ge-agent-factory/operations/)
  </Card>
  <Card title="One registry, three surfaces" icon="puzzle">
    Every `ge` command is a CLI command, an HTTP route, and an MCP tool — one
    operator registry, so they can never disagree. [Console & APIs →](/ge-agent-factory/reference/console-and-apis/)
  </Card>
</CardGrid>

## Try it in one terminal

```bash
mise run setup          # deps, toolchain, the `ge` CLI, daemon
mise run devex-smoke    # doctor → local mode → one validated canary workspace
mise run console        # operator UI: Pipeline · Fleet · Activity · Doctor
```

Nothing above touches the cloud — the build boundary keeps generation,
validation, and preview local until you choose to ship.
