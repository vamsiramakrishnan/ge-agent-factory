Get from a fresh clone to your first validated canary agent. Nothing touches
the cloud until you choose to ship — the everyday loop is entirely local.

<Steps>

1. **Set up the toolchain** (~5–10 min, one time).

   ```bash
   curl https://mise.run | sh   # once, if you don't have mise
   mise run setup               # JS deps + Python/uv toolchain + `ge` on PATH + daemon
   ```

2. **Initialize** (~30 s).

   ```bash
   ge init                      # discover config, write .ge.json
   ```

   Bare `ge` now prints a status board — mode, planes, and the next step.

3. **Prove the pipeline with a canary** (~5 min).

   ```bash
   ge devex smoke               # doctor → one generated, validated canary workspace
   ```

   The whole line — generate → validate → preview — runs on your machine
   against fixtures and simulators. See
   [the factory line](/ge-agent-factory/concepts/the-factory-line/) for what
   each stage produces.

4. **Build agents and watch them run.**

   ```bash
   ge mode local
   ge agents build --canary
   mise run console   # Overview · Pipeline · Interview · Fleet · Repair Queue · Runs · Readiness
   ```

   The factory emits a real workspace: ADK agent code, tools, fixture data,
   smoke tests, eval config, and a promotion packet. Every run is recorded —
   follow it live in **Runs**, or replay it event by event afterwards.

   Under the hood the build is one operator command — rendered here straight
   from the command registry, so it can't drift:

   <CommandCard id="agents.build.local" />

5. **Ship — when you're ready, and only then.**

   <Tabs>
     <TabItem label="Cloud Shell (guided)">
       Use the [Open in Cloud Shell tutorial](https://shell.cloud.google.com/?cloudshell_git_repo=https://github.com/vamsiramakrishnan/ge-agent-factory&cloudshell_workspace=installer&cloudshell_tutorial=installer/TUTORIAL.md)
       to clone the repo and run the installer step by step.
     </TabItem>
     <TabItem label="From a checkout">
       With `gcloud` authed against your project:

       ```bash
       export GEMINI_ENTERPRISE_APP_ID=projects/<num>/locations/global/collections/default_collection/engines/<app>
       CANARY=1 mise run bootstrap   # stand up the planes, prove one agent end to end
       ```

       Then follow [provision the platform](/ge-agent-factory/cookbooks/provision-the-platform/)
       and [Operations](/ge-agent-factory/operations/).
     </TabItem>
   </Tabs>

</Steps>

:::tip[Every surface is scriptable]
Every `ge` command accepts `--json`, and the same commands are HTTP routes
(the console) and MCP tools (model/harness callers) — one registry, three
surfaces. See the [CLI reference](/ge-agent-factory/reference/cli/).
:::
