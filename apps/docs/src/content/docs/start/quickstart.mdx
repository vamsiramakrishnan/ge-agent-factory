Finish with a proven agent workspace and the exact command that hands it to
Agent Engine and Gemini Enterprise. The default proof stays on this machine;
the handoff uses and mutates your Google Cloud project.

<Steps>

1. **Set up the toolchain** (~5–10 min, one time).

   ```bash
   curl https://mise.run | sh   # once, if you don't have mise
   mise run setup               # dependencies + `ge` on PATH + local task runner
   ```

2. **Capture a contract — or use the included starter.**

   ```bash
   ge capture                   # opens the guided interview in the console
   ```

   Finish the interview to create your own Enterprise Agent Contract. For a
   fast installation check, skip this step: `ge prove` automatically selects
   an included starter contract when nothing has been built yet.

3. **Prove one contract** (~5 min).

   ```bash
   ge prove                     # health check → agent workspace → evals and evidence
   ```

   This turns the captured or starter contract into code, tools, simulations,
   evals, and a validated workspace. It prints the exact workspace path and
   the next useful command.

4. **Read the contract the agent was compiled from.**

   ```bash
   ge state paths                            # shows where local state lands
   cd .ge/factory/workspaces/<workspace-id>  # the path `ge prove` printed
   head -60 mock_systems/usecase-spec.json
   ```

   The `behaviorContract` block is the agent's authority in plain sight:
   role, scope, tool intents, evidence requirements, escalation and refusal
   rules, golden evals. Every generated file in this workspace derives from
   it — see [the annotated contract](/ge-agent-factory/concepts/enterprise-agent-contract/).

5. **Run the proof yourself.**

   The generated eval dataset is in `agents-cli`'s format. Run it directly
   from the workspace:

   ```bash
   agents-cli eval run \
     --dataset tests/eval/datasets/ge_behavior_contract.json \
     --config tests/eval/eval_config.yaml
   ```

   The proof chain (evals, spec-to-code trace, verify-stage verdicts,
   promotion gate) is what stands between this workspace and a deploy:
   [Evals as proof](/ge-agent-factory/concepts/evals-as-proof/).

6. **Inspect proof progress in the console.**

   ```bash
   mise run console             # → http://localhost:18260
   ```

   Open **Runs** for the stage timeline and logs, select the agent to inspect
   its evidence, and open **Readiness** for the environment verdict.

   <details>
     <summary>Run another build from the lower-level command surface</summary>

     The command below is rendered from the shared command registry:

     <CommandCard id="agents.build.local" />
   </details>

7. **Hand off — when you're ready, and only then.**

   <Tabs>
     <TabItem label="Hand off a proven workspace">
       With the platform provisioned in your own GCP project
       ([Operations](/ge-agent-factory/operations/)):

       ```bash
       ge handoff agents-cli      # load data → deploy → register tools → publish
       ```

       The cloud consumes the exact workspace you proved locally — it never
       regenerates it. The proof and admission gates refuse the handoff if its
       evidence is not ready.
     </TabItem>
     <TabItem label="Stand up the platform first">
       Use the [Open in Cloud Shell installer](https://shell.cloud.google.com/?cloudshell_git_repo=https://github.com/vamsiramakrishnan/ge-agent-factory&cloudshell_workspace=installer&cloudshell_tutorial=installer/TUTORIAL.md),
       or from an authenticated checkout:

       ```bash
       export GEMINI_ENTERPRISE_APP_ID=projects/<num>/locations/global/collections/default_collection/engines/<app>
       mise run bootstrap-cloud      # stand up the cloud platform and run its readiness checks
       ```

       Then follow [Provision the platform](/ge-agent-factory/operations/provision-the-platform/).
     </TabItem>
   </Tabs>

</Steps>

:::tip[Every command is scriptable]
Every `ge` command accepts `--json`. The console and Model Context Protocol
(MCP) tools use the same command registry. See the [CLI
reference](/ge-agent-factory/reference/cli/).
:::
