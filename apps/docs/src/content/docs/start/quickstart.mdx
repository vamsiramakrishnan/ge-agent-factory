Run the whole layer once: compile an Enterprise Agent Contract into a real
agent workspace, inspect the contract and its proof artifacts, and see the
handoff command that gives the result to agents-cli / ADK / Gemini
Enterprise. Everything before the final step is local — no cloud
credentials, no side effects.

<Steps>

1. **Set up the toolchain** (~5–10 min, one time).

   ```bash
   curl https://mise.run | sh   # once, if you don't have mise
   mise run setup               # deps + Python/uv toolchain + `ge` on PATH + local daemon
   ```

2. **Initialize** (~30 s).

   ```bash
   ge init                      # discover config, write .ge.json
   ```

   Bare `ge` now prints a status board — local or cloud, platform state, and
   the next step. It defaults to **local**: nothing touches Google Cloud
   until you opt in.

3. **Compile and prove one contract** (~5 min).

   ```bash
   ge devex smoke               # doctor → compile one canary contract → validated workspace
   ```

   This takes one contract from the catalog through generate → validate —
   code, tools, simulations, evals, proof — and prints the workspace path.

4. **Read the contract the agent was compiled from.**

   ```bash
   ge state paths                            # shows where local state lands
   cd .ge/factory/workspaces/<workspace-id>  # the path `ge devex smoke` printed
   head -60 mock_systems/usecase-spec.json
   ```

   The `behaviorContract` block is the agent's authority in plain sight:
   role, scope, tool intents, evidence requirements, escalation and refusal
   rules, golden evals. Every generated file in this workspace derives from
   it — see [the annotated contract](/ge-agent-factory/concepts/enterprise-agent-contract/).

5. **Run the proof yourself.**

   The evalset is in `agents-cli`'s own format — the layer below, used
   directly, inside the generated workspace:

   ```bash
   agents-cli eval run --all
   ```

   The proof chain (evals, spec-to-code trace, verify-stage verdicts,
   promotion gate) is what stands between this workspace and a deploy:
   [Evals as proof](/ge-agent-factory/concepts/evals-as-proof/).

6. **Watch a compile live in the console.**

   ```bash
   ge agents build --canary     # compile one contract to the build boundary
   mise run console             # → http://localhost:18260
   ```

   Follow the run in **Runs** (live stage timeline + log tail), inspect the
   agent in **Fleet → Agent detail**, and check **Readiness** for the
   environment verdict. Under the hood the build is one registry-backed
   command — rendered here straight from the command registry, so it can't
   drift:

   <CommandCard id="agents.build.local" />

7. **Hand off — when you're ready, and only then.**

   <Tabs>
     <TabItem label="Ship a proven workspace">
       With the platform provisioned in your own GCP project
       ([Operations](/ge-agent-factory/operations/)):

       ```bash
       ge agents ship            # cloud runs load_data → deploy → register → publish
       ```

       The cloud consumes the exact workspace you proved locally — it never
       regenerates. The promotion gate blocks the ship if the proof isn't
       green.
     </TabItem>
     <TabItem label="Stand up the platform first">
       Use the [Open in Cloud Shell installer](https://shell.cloud.google.com/?cloudshell_git_repo=https://github.com/vamsiramakrishnan/ge-agent-factory&cloudshell_workspace=installer&cloudshell_tutorial=installer/TUTORIAL.md),
       or from an authenticated checkout:

       ```bash
       export GEMINI_ENTERPRISE_APP_ID=projects/<num>/locations/global/collections/default_collection/engines/<app>
       CANARY=1 mise run bootstrap   # stand up the planes, prove one agent end to end
       ```

       Then follow [Provision the platform](/ge-agent-factory/operations/provision-the-platform/).
     </TabItem>
   </Tabs>

</Steps>

:::tip[Every surface is scriptable]
Every `ge` command accepts `--json`, and the same commands are HTTP routes
(the console) and MCP tools — one registry, three surfaces. See the
[CLI reference](/ge-agent-factory/reference/cli/).
:::
