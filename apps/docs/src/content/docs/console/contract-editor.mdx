Two connected surfaces <a href="/ge-agent-factory/reference/glossary/#capture" title="The golden path's first verb — turning business intent into an Enterprise Agent Contract.">capture</a> intent and edit the resulting
[Enterprise Agent Contract](/ge-agent-factory/concepts/enterprise-agent-contract/): the
**<a href="/ge-agent-factory/reference/glossary/#interview" title="The guided, conversational flow where a business user (or agent) describes a use case and the factory turns it into a formal spec.">Interview</a>** view (conversational capture) and the **Spec Review** canvas
(structured editing and export).

## Interview — capture

An artifact-driven interview: a chat transcript pane on one side, a live
contract canvas on the other, and a document dropzone for grounding. You
describe the use case (or upload a BRD — a Business Requirements Document —
and let the interview read it); the interview agent asks what it still needs
and streams the contract into the canvas as it firms up — role, scope, tool
intents, workflow, evidence and escalation rules.

Use it when no contract exists yet. The result feeds straight into the
**<a href="/ge-agent-factory/reference/glossary/#pipeline" title="The user-facing, end-to-end path from business intent to a deployed agent — interview → spec → data → simulator → build → eval → preview → deploy — rendered as a plan you can read (ge pipeline plan) and watch (ge pipeline status).">Pipeline</a>** view for compilation. For step-by-step walkthroughs, see
[Capture from an interview](/ge-agent-factory/cookbooks/capture-from-interview/) and
[Capture from documents](/ge-agent-factory/cookbooks/capture-from-documents/).

## Spec Review — edit and export

The Spec Review canvas renders the contract half-by-half — the behavior half
(role, scope, rules, tool intents, workflow, golden evals) and the world
half (source systems, entities) — with field-level editing. It is also where
the contract's portable Markdown form is produced: the **Export <a href="/ge-agent-factory/reference/glossary/#okf--knowledge-bundle" title="OKF (Open Knowledge Format) is a portable, human-readable version of an agent's spec — a folder of plain Markdown files (called &quot;concepts&quot;) instead of one usecase-spec.json file.">OKF</a>** action
and the OKF (Open Knowledge Format) Knowledge Bundle preview
(`GET /api/interviews/<id>/okf` behind the scenes).

:::note
The OKF export API is fully wired; the canvas's export button is still
catching up. `curl` against `/api/interviews/<id>/okf` always works.
:::


## What it writes

- The contract: a catalog entry under `apps/factory/catalog/interview-specs/`
  (and `usecase-spec.json` in any <a href="/ge-agent-factory/reference/glossary/#workspace" title="The on-disk folder for one generated agent (.ge/factory/workspaces/workspace-id/), with a workspace.json manifest describing what's in it.">workspace</a> later compiled from it).
- Uploaded grounding documents, attached to the interview
  (`POST /api/interviews/<usecaseId>/documents`).
- The OKF bundle, on export.
