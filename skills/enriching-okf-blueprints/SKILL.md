# Enriching OKF Blueprints

You improve OKF blueprints through a classification-driven, patch-first enrichment workflow.

## Hard rules

1. Do not invent source systems, tools, entities, fields, or authority that are not in the OKF contract.
2. Do not mark a blueprint proven manually.
3. Do not edit generated docs directly.
4. Do not bypass proof gates or hash-bound proof checks.
5. Do not add generic evals.
6. Every new eval must have deterministic assertions.
7. Every action-tool eval must include `expected_state_delta` or `expected_no_mutation`.
8. Every high-risk or regulated spec must include adversarial and multi-turn coverage.
9. Every new fixture must be referentially consistent.
10. If domain expertise is insufficient, add `needs_expert_review` rather than fabricating certainty.

## Workflow

1. Read the shard manifest.
2. Run preflight commands.
3. Inspect each OKF spec and matching domain pack.
4. Generate reviewable patch JSON with `ge okf enrich generate`.
5. Dry-run apply with `ge okf enrich apply --patch <patch>`.
6. Apply patches only after reviewing allowed files and references.
7. Run `ge okf eval verify --spec <id>`.
8. Run `ge okf quality audit --spec <id> --fail-under <threshold>`.
9. Run proof commands when locally available.
10. Summarize changed specs, status movement, blockers, and command output.

## Harness notes

- Codex: run one shard per task/worktree and commit only source OKF/eval/fixture files plus `.enrichment` reports.
- Claude Code: use this skill with the shard prompt generated by `ge okf enrich prompt --harness claude`.
- Antigravity SDK: submit the generated prompt as a `harness.run` task, with this skill available in the workspace skill registry.

## Scripts

- `scripts/apply-shard.mjs` generates and applies reviewable enrichment patches for every spec in a shard.
- `scripts/run-quality-audit.mjs` runs `ge okf quality audit` for one spec or the full catalog.
- `scripts/verify-shard.mjs` verifies evals and quality thresholds for every spec in a shard.

Read `references/enrichment-rules.md` when you need the exact patch/eval requirements.
